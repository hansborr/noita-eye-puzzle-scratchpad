# Practice puzzle `six` — cube/Morse exact candidate

## Verdict

**Compelling exact candidate, pending external confirmation:**

> `CUBE IS A GREAT TOY MODEL OF NON-COMMUTATIVITY.`

The fixed-code portion after the first character is forced. The bounded model
also admits `FUBE IS A GREAT TOY MODEL OF NON-COMMUTATIVITY.` because the file
does not include the cube orientation immediately before the first observed
face. Both candidates re-encode all 139 face symbols exactly on each line. The
ordinary English word and sentence semantics select `CUBE`; the computation
alone leaves that one-bit/first-mark ambiguity visible.

This is deliberately called an **exact candidate**, not a maintainer-confirmed
decode. Initial cube orientation and roll-direction roles are swept, so exact
replay is a consistency gate rather than independent proof by itself. The full
search nevertheless cleared its matched null: `0/1024` direction-shuffled walks
produced any all-valid Morse candidate (`p_emp = 1/1025 = 0.00098`).

## Structural derivation

Treat labels `1..6` as cube faces with opposite pairs `(1,6)`, `(2,5)`, and
`(3,4)`. These pairs are not guessed from language:

- Every one of the 138 adjacent transitions in a line moves to one of the four
  adjacent faces. There are zero repeated-face and zero opposite-face jumps.
- Line 2 is a fixed relabeling of line 1:
  `1→2, 2→4, 3→6, 4→1, 5→3, 6→5`.
- Line 3 is another fixed relabeling:
  `1→3, 2→2, 3→6, 4→1, 5→5, 6→4`.
- Both relabelings preserve the three opposite pairs. The three lines are
  substitution-equivalent views of one walk, not three independent samples.

Sweep the 48 labeled cube orientations (24 rotations plus the two handed cube
nets retained because the fixture does not declare a net). In a compatible
orientation, the 139 observed top faces are generated by exactly three of the
four absolute roll directions. Assign those three directions in all six ways to
International Morse dot, dash, and inter-letter separator. The fixture's visible
spaces remain word separators and do not roll the cube.

One family of symmetry-equivalent cells reads the first line as standard Morse:

```text
-.-. ..- -... . / .. ... / .- / --. .-. . .- - / - --- -.-- /
-- --- -.. . .-.. / --- ..-. / -. --- -. -....- -.-. --- -- --
..- - .- - .. ...- .. - -.-- .-.-.-
```

That is `CUBE IS A GREAT TOY MODEL OF NON-COMMUTATIVITY.` The alternative
pre-stream orientation changes only the unobserved first roll mark, turning the
first `-.-.` (`C`) into `..-.` (`F`). Every later roll and Morse character is
the same.

## Gates and calibration

The Rust `cubemorse` instrument applies the following gates through the same
library functions used by its CLI and tests:

1. **Cube-walk gate:** every face must be adjacent to the current top face.
2. **Three-direction gate:** exactly three absolute roll directions must occur.
3. **Fixed Morse gate:** every delimited code must be a standard International
   Morse letter, digit, or punctuation code; empty codes are rejected.
4. **Exact replay:** re-encoding the candidate through the same cell must match
   every face and every supplied word length (`RoundTrip 139/139`).
5. **Matched null:** shuffle the derived roll commands while preserving total
   direction counts, total length, and all eight word-block lengths; rebuild a
   cube walk; rerun the complete orientation/role/Morse sweep. The registered
   1,024-trial run had zero valid-Morse survivors.

The self-test plants `CUBES MAKE ROLLS NON-COMMUTATIVE.`, encodes it through a
declared cube/Morse cell, recovers it verbatim with exact replay, and requires
all 32 matched control shuffles to remain negative.

## Reproduction

Calibrated Rust run on the first line:

```sh
sed -n '1p' research/data/practice-puzzles/six |
  cargo run --locked --quiet -- cubemorse --stdin --alphabet 123456 \
    --null-trials 1024 --seed 0x637562656d6f7273 --top 3
```

Run all three relabeled messages with the default 64-trial per-message null:

```sh
cargo run --locked -- cubemorse \
  --input-file research/data/practice-puzzles/six --alphabet 123456
```

Dependency-free Python reference for sharing/auditing the mechanics:

```sh
scripts/cubemorse_reference.py --null-trials 1024 \
  research/data/practice-puzzles/six
```

The earlier generic `solve --codec-search` record remains a valid scoped
negative for fixed base-6 grouping/delta codecs. It did not represent a rolling
cube with stateful Morse transduction, so it neither tested nor contradicts this
result.
