# 01 - hidden-base fixture and identifiability audit

**Status:** built 2026-07-06 on branch `agent/gak-unknown-base-recovery-01`.

## Scope

This is the first rung of `gak-unknown-base-recovery/`: a synthetic
known-plaintext fixture generator plus an identifiability-only audit for the
hidden shared base. It is not a solver for the real eyes and it is not a
language-scored ciphertext-only attack.

The audit deliberately separates two surfaces:

1. exact re-encryption of a supplied per-letter mapping; and
2. decomposition of that mapping as `perm(L) = B o sigma_L` under a hidden base
   `B` and bounded top-card-swap perturbations `sigma_L`.

So the report answers: if an exact re-encrypting key is already available, is the
hidden base uniquely identified by the bounded-decomposition model, or is there a
larger equivalent hidden-base class?

## Exact Cipher Convention

The instrument uses the same Lymm deck convention as `gak-swap-recover`.

Permutation arrays are images. Composition is:

```text
compose(p1, p2)[i] = p2[p1[i]]
```

Each message starts from the identity deck state. For each plaintext alphabet
letter `L`:

```text
state = compose(perm(L), state)
emit  ct_alphabet[state[0]]
perm(L) = B o sigma_L
```

Here `B` is the hidden shared base, and `sigma_L` is generated by at most `s`
top-card swaps `(0,k)`. Characters outside the plaintext alphabet pass through
and do not advance state, though the planted fixtures currently generate
alphabet-only messages.

## Landed Instrument

Library API:

- `plant_hidden_base_fixture`
- `audit_hidden_base_mapping`
- `run_hidden_base_identifiability_audit`
- `hidden_base_audit_self_test`

CLI:

```sh
cargo run --locked --bin noita-eye -- gak-hidden-base-audit \
  --n 11 \
  --num-swaps 2 \
  --messages 8 \
  --message-len 64 \
  --trials 8
```

The CLI plants deterministic random or affine hidden bases, generates several
identity-restart known-plaintext messages under one shared key, audits the
compatible hidden-base class, and prints the cipher convention in the report.

## Controls

The self-test runs through the same library surface as the CLI report:

| control | expected | measured outcome |
| --- | --- | --- |
| planted positive | accept | exact re-encryption, planted base retained |
| random full-permutation key | reject | exact re-encryption but no compatible hidden base |
| over-budget key attacked below budget | reject | exact re-encryption but no compatible hidden base |
| same over-budget key at true budget | accept | exact re-encryption, planted base retained |
| ciphertext-label shuffle | reject | bounded decomposition remains, exact re-encryption fails |

Measured default control output on 2026-07-06:

```text
hidden-base controls: PASS
  planted-positive: PASS expected=accept accepted=true exact=true bases=1 status=planted-base-unique
  random-full-permutation-key-null: PASS expected=reject accepted=false exact=true bases=0 status=no-compatible-base
  over-budget-key-null: PASS expected=reject accepted=false exact=true bases=0 status=no-compatible-base
  over-budget-true-budget-positive: PASS expected=accept accepted=true exact=true bases=1 status=planted-base-unique
  ciphertext-label-shuffle-null: PASS expected=reject accepted=false exact=false bases=1 status=round-trip-mismatch
```

## Measured Results

These are model-conditional synthetic-fixture measurements. Acceptance is exact
re-encryption only; the audit does not score language and does not touch the eye
corpus.

| command shape | observed result |
| --- | --- |
| `n=7, s=1, messages=8, len=64, trials=8` | `8/8` planted bases unique; compatible bases min/max `1/1`; sigma domain `7`; round-trip `512/512` on trial 0 |
| `n=11, s=2, messages=8, len=64, trials=8` | `8/8` planted bases unique; compatible bases min/max `1/1`; sigma domain `101`; round-trip `512/512` on trial 0 |
| `n=17, s=3, messages=8, len=64, trials=8` | `8/8` planted bases unique; compatible bases min/max `1/1`; sigma domain `3737`; round-trip `512/512` on trial 0 |
| `n=11, s=2, messages=1, len=1, trials=8` | `8/8` equivalent-base classes; compatible bases min/max `101/101`; one observed letter leaves `10` off-top base positions variable |
| `n=17, s=3, messages=1, len=1, trials=8` | `8/8` equivalent-base classes; compatible bases min/max `3737/3737`; one observed letter leaves `16` off-top base positions variable |

Interpretation: once a supplied exact mapping covers several observed letters,
the hidden base can become uniquely identified on these small random fixtures.
With only one observed letter, the base is not identifiable: every reachable
`sigma_L` yields an equivalent hidden base `B = perm(L) o sigma_L^-1`.

## Limits

- This is an identifiability audit after a mapping is supplied. It is not yet
  the unknown-base known-plaintext solver requested in task 02.
- The exact mapping in the positive fixtures comes from the plant; no claim is
  made that ciphertext alone identifies the mapping.
- The public-base substitution-first lesson is represented only as signal
  bookkeeping here: the report counts observed letters, identity-restart anchor
  coverage, unanchored observed letters, and unobserved letters. It does not run
  local search with a hidden base.
- The measured uniqueness is for random/affine small synthetic bases under
  `n in {7,11,17}` style surfaces. It should not be generalized to arbitrary GAK
  generators, larger `n`, or the real eyes.
